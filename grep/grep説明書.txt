# 電気設備・鉄道データで学ぶLinux基本コマンド解説書: grep

この技術解説書では、Linux/Unix環境においてテキストファイルの中から特定の文字列を含む行を検索・抽出できる「grep」コマンドについて、電気設備や電車のシステム・ログ管理を想定した具体例を交えて詳しく解説します。

---

## 1. grep コマンドの概要

`grep` は、ファイルまたは標準入力から渡されたテキストデータの中を検索し、指定したキーワード（パターン）が含まれている「行」を抜き出して画面に表示するコマンドです。
大量のログファイルから特定の機器のエラーだけを抽出したり、特定の列車のデータを絞り込んだりする際に必須となる、最も重要なコマンドの1つです。

### 基本文法
grep [オプション] "検索したい文字列" [ファイル名...]

---

## 2. 主なオプション

### 2.1 大文字と小文字を区別しない（-i）
デフォルトでは大文字と小文字を厳密に区別しますが、このオプションを付けると区別せずに検索します。
- 例: "error" を指定すると、"ERROR" や "Error" もヒットします。

### 2.2 検索キーワードに一致しない行を抽出する（-v）
指定した文字列が含まれて「いない」行（除外したいデータ）だけを反転して抽出します。不要なノイズログを取り除く際に便利です。

### 2.3 一致した行の前後を表示する（-A / -B / -C）
キーワードが見つかった行だけでなく、その周辺の行も一緒に表示します。トラブル前後のシステム挙動を確認するときに使います。
- -A n : 一致した行の「後（After）」n行も表示
- -B n : 一致した行の「前（Before）」n行も表示
- -C n : 一致した行の「前後（Context）」n行ずつを表示

### 2.4 一致した行数をカウントする（-c）
該当する行そのものを表示するのではなく、一致した「行の数」だけを出力します。

### 2.5 複数のディレクトリを再帰的に検索する（-r）
指定したディレクトリの中にあるすべてのファイルを対象に、中に含まれる文字列をまとめて検索します。

---

## 3. 電気設備・電車データを使った具体的な使用例

### 例1：変電所ログから「停電（blackout）」の記録だけを抜き出す
「substation.log」ファイルの中から、大文字小文字を問わず「blackout」が含まれる行を表示する場合：
grep -i "blackout" substation.log

### 例2：運行データから「普通列車（Local）」以外の行（急行や特急など）を抽出する
「train_list.txt」から、"Local" という文字列を含まない行だけを絞り込む場合：
grep -v "Local" train_list.txt

### 例3：ブレーカーの「遮断（trip）」ログと、その直前2行の挙動を確認する
「breaker.log」内で "trip" が発生した行と、その直前（Before）2行を表示して原因を調査する場合：
grep -B 2 "trip" breaker.log

### 例4：全路線のログフォルダから「架線異常（overhead_wire）」のログをまとめて一括検索する
「/var/log/railway」ディレクトリ以下のすべてのファイルから検索する場合：
grep -r "overhead_wire" /var/log/railway/

---

## 4. 他のコマンドとの実践的な組み合わせ（パイプライン連携）

grep は、他のコマンド（cat, wc, sort, uniq など）とパイプ（ | ）で繋ぐことで、より強力なデータ分析を行えます。

### 応用パターンA：特定のエラーが発生した回数を数える
【例：今日のスマートメーターのログから「通信エラー（timeout）」が何回発生したかを数える】
grep コマンドの「-c」オプションを使う方法と、「wc -l」を組み合わせる方法のどちらでも可能です。

grep -c "timeout" smart_meter.log
または
cat smart_meter.log | grep "timeout" | wc -l

### 応用パターンB：特定の異常が発生した「駅」のランキングを作る
【例：信号トラブル（signal_failure）のログから、発生回数の多い駅の順に並び替える】

cat signal.log | grep "signal_failure" | sort | uniq -c | sort -nr

1. cat signal.log で全体の信号ログを読み込む。
2. grep "signal_failure" で、信号トラブルに関する行だけを絞り込む（この時点で駅名が含まれる行だけが残る）。
3. sort で駅名ごとに並び替えて密集させる。
4. uniq -c で、各駅でトラブルが何回発生したかをカウントする。
5. sort -nr で、発生回数の多い順（数値の大きい順）に並び替える。

これにより、どの駅の信号機で最もトラブルが多発しているかを特定し、優先的に電気設備の点検を行う計画を立てることができます。
