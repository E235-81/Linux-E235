# 電気設備・鉄道データで学ぶLinux基本コマンド解説書: wc, sort, uniq

この技術解説書では、Linux/Unix環境においてテキストデータの解析や加工によく使用される3つの基本コマンド（wc, sort, uniq）について、電気設備や電車のデータを用いた具体例を交えて詳しく解説します。

---

## 1. wc コマンド (Word Count)

### 概要
wc は、ファイルまたは標準入力から渡されたテキストの「行数」「単語数」「バイト数（文字数）」をカウントするためのコマンドです。データの総件数を調べたり、ログの規模を確認したりする際に使用します。

### 基本文法
wc [オプション] [ファイル名...]

### 主なオプション
- -l : 行数のみをカウントして出力します。（実務で最も頻繁に使われます）
- -w : 単語数のみをカウントします。
- -c : バイト数のみをカウントします。

### 電気設備・電車の例
1. 「変電所の異常ログ一覧（error_log.txt）」の総件数（行数）を調べる場合：
   wc -l error_log.txt
   # 出力例: 45 error_log.txt （45件の異常ログが記録されていることを意味します）

2. 「路線の全駅名リスト（stations.txt）」の駅数（行数）を調べる場合：
   wc -l stations.txt
   # 出力例: 29 stations.txt （全29駅あることを意味します）


---

## 2. sort コマンド

### 概要
sort は、テキストファイルの内容を行単位で並び替える（ソートする）コマンドです。デフォルトでは文字コード順（辞書順）ですが、オプションで行の特定部分を数値として並び替えることもできます。

### 基本文法
sort [オプション] [ファイル名...]

### 主なオプション
- -n : 文字列ではなく「数値」として大小を比較し並び替えます。
- -r : 並び替えの順序を逆順（降順、大きい順）にします。
- -k : 特定の列（フィールド）を指定してソートします（デフォルトは空白区切り）。
- -t : 列の区切り文字（セパレータ）を指定します（例：CSVなら -t ','）。

### 電気設備・電車の例
1. 「スマートメーターの電流計測データ（amps.txt）」を数値の小さい順に並び替える：
   sort -n amps.txt
   # 2A, 5A, 15A, 20A のように正しく数値順に並びます。-n がないと 15A が 2A より前に来てしまいます。

2. 「各駅の乗車人員数リスト（riders.txt）」を、人数の多い順（降順）に並び替える：
   sort -nr riders.txt

3. カンマ区切りの「電車運行ダイヤCSV（timetable.csv）」の第2列（発車時刻）を基準にソートする：
   sort -t ',' -k 2,2 timetable.csv

## 3. uniq コマンド (Unique)

### 概要
uniq は、入力されたテキストから重複する行を取り除く（または重複を抽出する）コマンドです。

注意点：
uniq コマンドは「隣り合った行（連続する行）」しか重複判定できません。そのため、ファイル全体の重複を完全に排除したい場合は、必ず事前に sort コマンドで並び替えておく必要があります。

### 基本文法
uniq [オプション] [入力ファイル]

### 主なオプション
- -c : 各行が何回連続して出現したか（カウント数）を行の先頭に付与します（データ集計の定番）。
- -d : 重複している行（2回以上現れる行）だけを出力します。

### 電気設備・電車の例
1. 「通過した電車の種別ログ（train_types.txt）」から、連続する重複をまとめる：
   uniq train_types.txt
   # 「普通、普通、普通、急行」というログが「普通、急行」にまとまります。

2. 「スマートメーターの電圧低下警告ログ（low_voltage.txt）」で、各時間帯に何回警告が出たか数える：
   uniq -c low_voltage.txt
   # 出力例: 
   #   5 10:00 （10:00の警告が5回連続した）
   #   1 11:00 （11:00の警告が1回発生した）

## 4. 実践的な組み合わせ（パイプライン連携）

これら3つのコマンドは、パイプ（ | ）を使って組み合わせることで、強力なデータ分析ツールになります。

### パターンA：重複のない種類（件数）を特定する
【例：今日、トラブルが発生した「変電所の場所（機器ID）」が何箇所あるかを調べる】
同じ変電所で何度もエラーが出ている場合、重複をカットして「箇所の数」だけを数えます。

sort error_devices.txt | uniq | wc -l

1. sort で機器IDを並べ替え、同じIDを隣り合わせにする。
2. uniq で重複したIDを1つにまとめる（これで発生した場所一覧になります）。
3. wc -l でその行数をカウントする（エラーが出た変電所の総箇所数がわかります）。


### パターンB：データの出現頻度ランキングを作る
【例：どの駅で最も多く「自動改札機の詰まり（エラー）」が発生しているかランキングを作る】

cat gate_errors.log | sort | uniq -c | sort -nr

1. cat gate_errors.log でエラーが発生した駅名リストを読み込む。
2. sort で駅名をアルファベット順（または50音順）に並び替え、同じ駅名を密集させる。
3. uniq -c で、各駅名が何回登場したか（エラー回数）をカウントする。
4. sort -nr で、カウントされた「エラー回数（数値）」を「大きい順（逆順）」に並び替える。

結果として、「渋谷駅: 120回」「新宿駅: 95回」のように、トラブルの多い駅が上から順番に表示されるため、優先的にメンテナンスすべき設備をひと目で特定できます。
