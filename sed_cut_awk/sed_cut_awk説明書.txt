# 電気設備・鉄道データで学ぶLinux基本コマンド解説書: sed, cut, awk

この技術解説書では、Linux/Unix環境においてテキストデータの抽出や高度な編集・加工を行うための3つの重要コマンド（sed, cut, awk）について、電気設備や電車のシステム・ログ管理を想定した具体例を交えて詳しく解説します。

---

## 1. sed コマンド (Stream Editor)

### 概要
sed は、テキストの「置換」「削除」「挿入」などを非対話的（画面でファイルを開かずに）に行うストリームエディタです。特に、ファイル内の特定の文字列を一括で置き換える際に非常によく使われます。

### 基本文法
sed [オプション] 'スクリプト（指示）' [ファイル名]

### 主な使い方
- 's/検索文字列/置換文字列/g' : 該当する文字列をすべて置換する（最頻出）
- 'd' : 条件にマッチした行を削除する

### 電気設備・電車の例
1. ログファイル内の「変電所A（substation_A）」という名称を「変電所1（substation_1）」にすべて書き換える：
   sed 's/substation_A/substation_1/g' power.log

2. 運行データ（train.txt）から「回送（Out_of_service）」と書かれた行をすべて削除して表示する：
   sed '/Out_of_service/d' train.txt


---

## 2. cut コマンド

### 概要
cut は、テキストの各行から「特定の列（フィールド）」や「特定の文字位置」だけを切り出す（抽出する）ためのコマンドです。CSV形式のデータなど、区切り文字で整理された単純なデータから特定の項目だけを集めたいときに最適です。

### 基本文法
cut [オプション] [ファイル名]

### 主なオプション
- -d '区切り文字' : 列を区切る記号を指定します（デフォルトはタブ）。
- -f 列番号 : 抽出する列を指定します（複数指定時は「2,4」や「2-4」のように指定）。

### 電気設備・電車の例
1. カンマ区切りの「駅情報CSV（stations.csv）」から、2列目（駅名）だけを切り出す：
   cut -d ',' -f 2 stations.csv

2. スペース区切りの「スマートメーター電流データ（amps.txt）」から、1列目（時間）と3列目（電流値）を切り出す：
   cut -d ' ' -f 1,3 amps.txt


---

## 3. awk コマンド

### 概要
awk は、単なるテキスト抽出を超えて、高度なパターンマッチング、条件分岐、計算、レポート作成などが可能な「プログラミング言語」に近い強力なコマンドです。空白の数が不規則なログファイルの処理や、数値の計算を伴うデータ集計で真価を発揮します。

### 基本文法
awk [オプション] 'パターン { アクション }' [ファイル名]

### 主な特徴と変数
- $1, $2, $3... : 1列目、2列目、3列目…のデータを表します（$0 は行全体）。
- -F '区切り文字' : 列の区切り文字を指定します（省略時は自動的に「任意の連続する空白」を区切りとみなします）。

### 電気設備・電車の例
1. 運行遅延ログ（delay.log）の3列目に書かれた「遅延分数」が「10分以上」の行だけを表示する：
   awk '$3 >= 10 { print $0 }' delay.log

2. 変電所の出力データ（power.txt）の2列目（電力値 kW）の「合計値（総合電力量）」を計算して出力する：
   awk '{ sum += $2 } END { print "総電力量:", sum, "kW" }' power.txt
   # 各行の2列目を sum に足していき、最後に（END）合計を表示します。


---

## 4. これまでのコマンドを組み合わせた実践例

これまで学んだコマンド（grep, sort, uniq, wc など）に sed, cut, awk を組み合わせることで、複雑な管理業務を1行のコマンド（ワンライナー）で自動化できます。

### 応用パターンA：不規則なログから特定のデータだけをきれいに抜き出す
【例：空白の数がバラバラな設備ログから、エラーの起きた「時間」と「機器ID」だけをCSV形式で保存する】
※ログの形式：[2026-07-02 10:00:23]   ERROR     Device_ID_402   Voltage_Drop

cat facility.log | grep "ERROR" | awk '{ print $2 "," $4 }' > error_list.csv

1. grep "ERROR" で、エラーの行だけを絞り込む。
2. awk '{ print $2 "," $4 }' が、連続する空白を自動調整しつつ、2列目（時間）と4列目（機器ID）をカンマで繋いで出力する。
3. リダイレクト（ > ）でCSVファイルに保存する。

### 応用パターンB：特定の路線で「遅延が多発している列車種別」のランキングを作る
【例：運行管理データから、5分以上の遅延を起こした「特急」や「急行」などの種別をカウントし、多い順に並べる】
※データの形式：[列車番号],[種別],[行先],[遅延分数]

cat railway.csv | cut -d ',' -f 2,4 | awk -F ',' '$2 >= 5 { print $1 }' | sort | uniq -c | sort -nr

1. cut -d ',' -f 2,4 で、2列目（種別）と4列目（遅延分数）だけを切り出す（処理を軽くするため）。
2. awk -F ',' '$2 >= 5 { print $1 }' で、切り出したデータの2列目（遅延分数）が5以上の行を対象に、1列目（種別）だけを出力する。
3. sort | uniq -c で、遅延した種別（特急、快速など）ごとに回数をカウントする。
4. sort -nr で、回数の多い順に並び替える。

これによって、「どの列車種別がダイヤ乱れの影響を受けやすいか」を統計的にひと目で把握できるようになります。
