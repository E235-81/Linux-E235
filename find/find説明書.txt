# 電気設備・鉄道データで学ぶLinux基本コマンド解説書: find

この技術解説書では、Linux/Unix環境においてファイルやディレクトリを強力な条件指定で検索できる「find」コマンドについて、電気設備や電車のシステム・ログ管理を想定した具体例を交えて詳しく解説します。

---

## 1. find コマンドの概要

`find` は、指定したディレクトリ（フォルダ）の下層から、ファイル名、ファイルサイズ、更新日時、ファイルの種類などの様々な条件を指定して、目的のファイルやディレクトリを検索するコマンドです。
大量のログファイルや機器データの中から、特定の条件に一致するファイルを抽出する際に必須となります。

### 基本文法
find [検索を開始するディレクトリ] [検索条件] [アクション]

※ 検索条件を省略した場合は、指定したディレクトリ以下のすべてのファイルが表示されます。

---

## 2. 主な検索条件（オプション）

### 2.1 ファイル名で探す（-name / -iname）
指定した文字列に一致するファイル名を探します。ワイルドカード（*）を使うことで、「〜で始まる」「〜で終わる」といった曖昧な検索が可能です。
- -name : 大文字と小文字を区別する
- -iname : 大文字と小文字を区別しない

### 2.2 ファイルの種類で探す（-type）
ファイルかディレクトリかを指定して検索します。
- -type f : 通常のファイルのみを対象にする
- -type d : ディレクトリ（フォルダ）のみを対象にする

### 2.3 更新日時で探す（-mtime）
ファイルが最後に修正（更新）されたシステム日時を基準に検索します。
- -mtime -n : n日以内（現在から24時間×nの時間内）に更新されたファイル
- -mtime +n : n日より前（24時間×nよりも昔）に更新されたファイル

### 2.4 ファイルサイズで探す（-size）
ファイルの容量を基準に検索します。
- -size +n : nより大きいファイル（k: キロバイト、M: メガバイト、G: ギガバイト）
- -size -n : nより小さいファイル

---

## 3. 電気設備・電車データを使った具体的な使用例

### 例1：特定の駅のデータファイルをファイル名で探す
「/data/stations」ディレクトリ以下から、ファイル名に「shinjuku」が含まれるファイルを探す場合：
find /data/stations -name "*shinjuku*"

### 例2：今日（24時間以内）更新された変電所のログファイルを探す
「/var/log/substation」以下から、1日以内（-mtime -1）に更新された「ファイル（-type f）」を探す場合：
find /var/log/substation -type f -mtime -1

### 例3：容量が100MBを超えている巨大な運行ダイヤログを探す
ディスク容量を圧迫している古いデータや異常ログを見つけるために、100メガバイトより大きいファイル（-size +100M）を探す場合：
find /data/timetable -type f -size +100M

---

## 4. 実践的な応用（検索結果に対する処理）

find コマンドの真価は、検索して見つかったファイルに対して、そのまま別のコマンドを実行できる点にあります。これには「-exec」オプションを使用します。

### 応用パターンA：見つかったファイルの行数を一括で数える
【例：今月更新されたすべての「エラーログ」の行数を wc コマンドでまとめてカウントする】
find /var/log/electrical -name "*.log" -mtime -30 -exec wc -l {} +

- {} の部分に、find で見つかったファイル名が自動的に代入されます。
- 末尾の +（または \;）は、-exec オプションの終了を意味する決まり文句です。

### 応用パターンB：古いログファイルを自動的に削除・移動する
【例：半年前（180日前）より古い、容量の大きいバックアップファイルを自動で削除する】
find /backup/train_data -type f -mtime +180 -exec rm {} \;

注意：rm などの削除コマンドを組み合わせる場合は、事前に exec 部分を外して find だけで実行し、意図しないファイルが含まれていないか確認することを強く推奨します。
