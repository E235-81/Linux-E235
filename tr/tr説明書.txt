# 電気設備・鉄道データで学ぶLinux基本コマンド解説書: tr

この技術解説書では、Linux/Unix環境において文字の置換や削除、連続する文字の集約を効率的に行う「tr」コマンドについて、電気設備や電車のシステム・データ管理を想定した具体例を交えて詳しく解説します。

---

## 1. tr コマンドの概要

`tr` は「translate」の略で、標準入力から受け取ったテキストデータに対して、指定した「文字の置換（置き換え）」や「文字の削除」を文字単位で行うコマンドです。
find や grep とは異なり、引数に直接ファイル名を指定して実行することができないため、通常はパイプ（ | ）やリダイレクト（ < ）を使ってデータを受け渡します。

### 基本文法
tr [オプション] "検索する文字セット" ["置換する文字セット"]

---

## 2. 主な機能とオプション

### 2.1 文字の置換（オプションなし）
指定した文字を別の文字に置き換えます。1文字ずつの対応関係で置換されるため、大文字から小文字への一括変換などに重宝します。

### 2.2 文字の削除（-d）
指定した文字をテキスト内からすべて取り除きます（delete）。
- -d : 指定した文字を削除する

### 2.3 連続する重複文字を1文字にまとめる（-s）
同じ文字が何個も連続している場合、それを1つの文字に集約します（squeeze）。スペースやタブで体裁が崩れたテキストを整形する際に便利です。

---

## 3. 電気設備・電車データを使った具体的な使用例

### 例1：小文字の路線コードを大文字に統一する
システムによって「tokyo_line」のように小文字で記録されたデータを、すべて大文字（TOKYO_LINE）に変換する場合：
echo "tokyo_line" | tr "a-z" "A-Z"
# 出力例: TOKYO_LINE
# ※「a-z」はアルファベットの小文字すべて、「A-Z」は大文字すべてを指す範囲指定です。

### 例2：CSVファイルから「電流の単位（A）」を削除して数値だけにする
「amps_data.txt」の中に「15A, 20A, 30A」と書かれているデータから、一括で「A」を取り除いて計算しやすくする場合：
cat amps_data.txt | tr -d "A"
# 出力例: 15, 20, 30

### 例3：ログ内の無駄な連続スペースを1つにまとめる
変電所の出力ログ（power.log）の体裁が崩れ、値の間に余分な空白が大量に入ってしまっている場合、スペース（" "）を1つに集約します：
cat power.log | tr -s " "
# 「10:00    変電所A    正常」というデータが「10:00 変電所A 正常」に整形されます。

### 例4：改行コードをカンマに変換して、縦のリストを横一列にする
「stations.txt」に縦並びで記録されている駅名リストを、カンマ区切りの横並びデータに変換する場合：
cat stations.txt | tr "\n" ","
# ※「\n」は改行を意味する特殊文字です。

---

## 4. 他のコマンドとの実践的な組み合わせ（パイプライン連携）

tr は、データの「前処理」として他のコマンドと組み合わせることで、データの集計精度を劇的に向上させます。

### 応用パターンA：区切り文字を整形して目的の列を正しく抽出する
【例：空白の数がバラバラな運行ログから、3列目の「遅延時間」だけを抜き出してソートする】

cat train_delay.log | tr -s " " | sort -k 3 -n

1. cat train_delay.log でログを読み込む。
2. tr -s " " で、列と列の間に挟まっているランダムな数のスペースを、すべて「半角スペース1個」に統一する（これにより、後ろのコマンドが正しく列を認識できるようになります）。
3. sort -k 3 -n で、整形されたデータの3列目（遅延時間）を基準に数値順でソートする。

### 応用パターンB：表記のゆれ（大文字・小文字）を統一してエラーランキングを作る
【例：ログ内に「ERROR」「Error」「error」が混在している状態から、正しく件数を集計する】
※そのまま集計すると、別のエラーとしてカウントされてしまいます。

cat substation.log | tr "A-Z" "a-z" | grep "error" | wc -l

1. tr "A-Z" "a-z" で、すべての文字をいったん小文字に統一する（表記のゆれを消す）。
2. grep "error" で、小文字になった「error」の行だけを確実に引っかける。
3. wc -l で、システム全体の総エラー件数を正確にカウントする。
